Black Hole WareHouse - GitHub Pages Compatibility

Compatibility Status: The Black Hole WareHouse documentation site is fully compatible with GitHub Pages.

Deployment Instructions

To deploy this site to GitHub Pages:

  1. Create a new GitHub repository
  2. Upload all files from the black-hole-warehouse directory to the repository
  3. Go to repository Settings → Pages
  4. Under "Source", select "main" branch
  5. Click "Save"
  6. Your site will be published at https://[username].github.io/[repository-name]/

Jekyll Configuration

The site uses the following Jekyll configuration:

title: Black Hole WareHouse
description: Documentation for Black Hole V1.0 - The ultimate WiFi security testing device
baseurl: ""
url: ""

# Build settings
markdown: kramdown
highlighter: rouge
theme: jekyll-theme-minimal
plugins:
  - jekyll-feed
  - jekyll-seo-tag
Note: If deploying to a subdirectory (e.g., username.github.io/black-hole-warehouse), update the baseurl in _config.yml to "/black-hole-warehouse".

  • ✅ All markdown files include proper front matter

Local Testing

To test the site locally before deployment:

# Install Jekyll and Bundler
gem install jekyll bundler

# Navigate to the site directory
cd black-hole-warehouse

# Install dependencies
bundle install

# Start local server
bundle exec jekyll serve

The site will be available at http://localhost:4000.
